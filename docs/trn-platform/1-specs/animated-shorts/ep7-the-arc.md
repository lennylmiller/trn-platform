## SOURCE MATERIAL

### The Problem (20 seconds)
AI-assisted "vibe coding" lets developers ship at 10x speed. By month 3, the code fights back. Gartner: 45% of AI-generated code has security vulnerabilities. Defect rates increase 2,500%. The $4.7 billion vibe coding market creates — but it doesn't sustain. This is the Spaghetti Point.

### The Thesis (25 seconds)
Software complexity resolves in layers — like a microscope with zoom levels. Layer 1: Components (small, testable, domain-scoped). Layer 2: Workflow Stories (user journeys composing components). Layer 3: Pages (thin orchestrators). Layer 4: Customer Forks (per-tenant configuration). Between them: connective tissue — machine-readable catalogs and schemas that make the architecture traversable by both humans and AI. The rule: never skip a layer. This is Layers of Resolution.

### The Proof (25 seconds)
PFM-Platform proved it at scale: 148 components across 11 financial domains. 33 npm packages. Zero cross-domain imports. The pattern held at every scale — from a single hook to an entire domain, the same structure repeats. At 148 components, the pattern became undeniable: consistent → extractable → automatable → productizable.

### The Pipeline (25 seconds)
lar-platform: 10 sequential AI prompts that decompose a domain into a platform. Domain discovery → entity extraction → use cases → workflows → component catalog → pages catalog → test strategy → infrastructure → Storybook → SaaS shell. Each prompt builds on the last. The AI is constrained by the architecture, not freed from it. That's the opposite of vibe coding. The pipeline was tested on a trip planner: plain English in, 8 domains and 18 packages out, in 6 hours.

### The Meta-Proof (30 seconds)
Two platforms prove the pipeline works: TP-Platform (8 domains, Supabase, 3 layers) and TRN-Platform (4 domains, SQL Server + Express, 4 layers). Same pipeline, different backends, different domains.

Here's the twist: TRN-Platform has a compositions system for delivering presentations. Narrative blocks, flow blocks that execute live SQL, nested compositions. The Bridge Builder presentation — the one telling this very story — runs on TRN-Platform. The platform generated by the pipeline is presenting the pipeline's story. There's even a demo where it queries its own database to show the row for the composition currently being displayed. The presentation IS data.

One platform is a project. Two is a pattern. A platform presenting its own origin is a proof.

### The Product (20 seconds)
Bridge Builder takes the pipeline and makes it a product. Describe your domain in plain English. Get a production-grade platform back. Not a rewrite — a decomposition. The spaghetti unwinds into architecture. Six POCs building toward a SaaS that transforms vibe-coded apps into platforms where features are spoken into existence.

Vibe coding creates. Platform architecture sustains. Bridge Builder bridges the gap.
