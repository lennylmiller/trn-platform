# Episode 0: "The Twist" (Series Teaser)

**Duration:** 90 seconds
**Purpose:** Hook the audience. Establish the self-referential premise.
**Maps to:** Full arc (teaser for all 6 acts)

---

## NOTEBOOKLM BRIEF

Create a 90-second audio teaser where two hosts set up a mystery. They describe a training platform that runs live SQL, streams results, and walks people through interactive stories. Then one host realizes something: the platform they're describing was built by the very pipeline the presentation is about. The tone is curious, building to a genuine "wait, what?" moment. End with the tagline: "The Bridge Builder Story — a platform that presents its own origin."

---

## SOURCE MATERIAL

There is a training platform called TRN-Platform. It delivers interactive training sessions through "compositions" — stories made of narrative blocks (prose), flow blocks (live executable demos), and note blocks (technical deep-dives).

During a presentation, the system executes real SQL queries against a live SQL Server database. Server-Sent Events stream the results in real time. Presenters can pause between steps, discuss what happened, and resume.

Here is the twist: this platform was generated by a 10-prompt AI pipeline called lar-platform. The pipeline takes a domain description in plain English and produces a full, architecturally sound monorepo — typed packages, API routes, query hooks, UI components, Storybook stories, and yes, the composition system itself.

So when you watch a Bridge Builder presentation, you are watching a platform that was created by the pipeline the presentation is describing. The composition blocks, the flow execution engine, the SQL step library — all of it came from the pipeline. The platform is presenting its own origin story.

And if you run the "Composition Self-Query" demo during the presentation, you can watch the platform query the database row for the very composition it is currently displaying. The abstraction collapses into data.
