# UC-COMP-7: Reorder Blocks

**Actor:** Author | **Tier:** Core | **Workflow:** WF3 Author Story

## Precondition
A composition is open in the editor with two or more blocks.

## Trigger
Drag block handle to reorder.

## Main Flow
1. Author grabs the drag handle on a CompositionBlockCard
2. Author drags the block to the desired position (up or down)
3. System resequences all blocks and updates seq values
4. System sends PUT /api/v2/compositions/:id/blocks with new order
5. Blocks display in the new sequence

## Alternate Flows
- **AF-1:** Single block -- Drag handles are visible but non-functional when only one block exists
- **AF-2:** Drop in same position -- No change is persisted

## Postcondition
Blocks are resequenced in the new order; seq values are contiguous starting from 1.
