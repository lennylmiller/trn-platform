# UC-COMP-8: Edit Block Properties

**Actor:** Author | **Tier:** Core | **Workflow:** WF3 Author Story

## Precondition
A composition is open in the editor with at least one block.

## Trigger
Click edit button on a block card.

## Main Flow
1. Author clicks the edit action on a CompositionBlockCard
2. System opens the BlockPropertiesPanel for the selected block
3. Author edits heading text
4. Author edits content/technical_content
5. Author edits presenter_notes
6. For flow blocks: author can change flow via picker
7. System sends PUT /api/v2/compositions/:id/blocks/:blockId

## Alternate Flows
- **AF-1:** Cancel edit -- Author closes panel without saving; no changes persisted
- **AF-2:** Validation error -- Empty required fields show inline validation messages

## Postcondition
Block properties are updated in the database and reflected in the editor.
