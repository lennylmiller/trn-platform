# UC-EXEC-7: View Results

**Actor:** System/User | **Tier:** Core | **Workflow:** WF2 Present Flow

## Precondition
Flow execution is in progress or has completed. The SSE connection is active via `useSSE` hook.

## Trigger
Step execution produces output -- `step:output` events arrive via the SSE stream.

## Main Flow
1. During execution, SSE events stream to the client via `useSSE` hook
2. `step:output` events are processed by `useExecutionState` and appended to the output buffer
3. `ConsoleDrawer` (bottom panel) renders output lines in real time with auto-scroll
4. For `sql` type steps: results are parsed and displayed in `SqlResultsModal` as a formatted data table via `useExecuteSql`
5. `EventStream` component shows a timeline of all execution events (start, output, pause, complete, error) with timestamps
6. After `execution:complete`, all output remains available for review -- user can scroll through `ConsoleDrawer` and re-open `SqlResultsModal`

## Alternate Flows
- **AF-1:** stderr output -- lines rendered in red monospace font in the `ConsoleDrawer`
- **AF-2:** Large output -- `ConsoleDrawer` auto-scrolls to latest line, user can scroll up to review; virtualized list for performance
- **AF-3:** SQL error -- `SqlResultsModal` shows the error message with the failing query highlighted

## Postcondition
All execution output is visible and reviewable in the console drawer, SQL results modals, and event stream timeline.
