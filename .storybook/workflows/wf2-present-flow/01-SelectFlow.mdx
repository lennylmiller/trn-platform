# UC-FLOW-1: Select Flow

**Actor:** User | **Tier:** Core | **Workflow:** WF2 Present Flow

## Precondition
User is authenticated and at least one flow exists in the system with configured steps.

## Trigger
User navigates to the Flows tab to select a flow for presentation.

## Main Flow
1. System fetches flows from `GET /api/v2/flows` via the `useFlows` hook
2. `FlowListTab` renders each flow as a `FlowCard` component showing: flow name, step count badge, description preview
3. User reviews available flows -- cards are sorted by last modified date
4. User clicks the "Present" button on the desired flow card
5. System navigates to the presentation view for the selected flow

## Alternate Flows
- **AF-1:** No flows exist -- empty state with illustration and "Create your first flow" call-to-action button
- **AF-2:** API error -- MUI `Alert` with severity "error" and a Retry button

## Postcondition
User has selected a flow and the system is ready to enter presentation mode.
