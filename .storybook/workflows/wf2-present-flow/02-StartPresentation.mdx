# UC-FLOW-9: Start Presentation

**Actor:** User | **Tier:** Core | **Workflow:** WF2 Present Flow

## Precondition
A flow has been selected from the flow list. The flow has at least one step with `visible_in_execution` set to true.

## Trigger
User clicks "Present" on a flow card, navigating to `FlowRunPage`.

## Main Flow
1. System fetches the full flow detail from `GET /api/v2/flows/:id` via the `useFlowPresenter` hook
2. `FlowRunPage` enters presentation mode -- full-screen layout with step details center stage
3. System filters steps to only those with `visible_in_execution = true`
4. First visible step is displayed with: label (large heading), type chip (colored), description text
5. Presenter notes section is available as a collapsible `Accordion` below the step details
6. Step progress indicator shows "Step 1 of N" in the top bar
7. "Continue" button is prominently displayed to begin execution

## Alternate Flows
- **AF-1:** No visible steps -- "No visible steps in this flow" message displayed with a link back to the flow editor
- **AF-2:** Flow not found -- 404 error page with navigation back to flow list

## Postcondition
Presentation mode is active and the first visible step is displayed, ready for the user to begin execution.
