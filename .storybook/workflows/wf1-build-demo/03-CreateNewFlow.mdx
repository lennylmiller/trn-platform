# UC-FLOW-2: Create Flow

**Actor:** User | **Tier:** Core | **Workflow:** WF1 Build Demo

## Precondition
User is on the Flows tab with create permissions.

## Trigger
User clicks the "+ New Flow" button in the `FlowListTab` toolbar.

## Main Flow
1. User clicks "+ New Flow" button
2. System opens a create dialog with name and description fields
3. User enters a flow name (required, validated by `FlowCreateSchema`)
4. User optionally enters a description
5. User clicks Save
6. System validates with `FlowCreateSchema` (Zod)
7. System calls `POST /api/v2/flows` via the `useCreateFlow` mutation hook
8. On success: dialog closes, `useFlows` query invalidated, new flow appears in the list with 0 steps

## Alternate Flows
- **AF-1:** Empty name -- validation error prevents save, inline error shown below name field
- **AF-2:** API error -- MUI `Alert` with error message and Retry action

## Postcondition
A new empty flow exists in the `qc_training.flows` table and is visible in the flow list with a step count of 0.
