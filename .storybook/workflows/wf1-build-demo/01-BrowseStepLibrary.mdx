# UC-STEP-1: Browse Step Library

**Actor:** User | **Tier:** Core | **Workflow:** WF1 Build Demo

## Precondition
User is authenticated and has access to the Steps tab. The `qc_training.step_library` table contains at least one step.

## Trigger
User navigates to the Steps tab.

## Main Flow
1. System fetches steps from `qc_training.step_library` via the `useSteps` hook
2. `StepListTab` renders steps grouped by category using the `useGroupedSteps` hook
3. Categories displayed as filterable chips: **setup**, **scenario**, **sync**, **verify**, **utility**
4. Each step card shows: label, type chip (colored via `STEP_TYPE_COLORS` constant), category chip, and description
5. User clicks a category chip to filter -- `useStepFilters` updates the active filter
6. User types in the search field to filter by label or description (debounced 300ms)
7. System re-renders the list showing only matching steps
8. Category enum values come from `StepCategorySchema` (Zod enum)

## Alternate Flows
- **AF-1:** No steps match filters -- "No steps match" empty state with illustration and clear-filters button
- **AF-2:** API fetch fails -- MUI `Alert` component with severity "error" and a Retry button that re-invokes the query

## Postcondition
User sees a filtered, searchable list of available steps grouped by category.
