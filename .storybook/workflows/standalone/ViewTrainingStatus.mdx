# UC-EXEC-8: View Training Status

**Actor:** User | **Tier:** Core | **Workflow:** Standalone

## Precondition
Training database (qc_core) is accessible.

## Trigger
Navigate to status view.

## Main Flow
1. System queries GET /api/v2/execute/status
2. System displays status cards (members, PCP, claims, referrals)
3. Each card shows count and key data
4. Data presence indicator (green check if data exists)

## Alternate Flows
- **AF-1:** Database inaccessible -- Error banner shown with connection details
- **AF-2:** Empty database -- All counts show 0 with a "No data loaded" indicator

## Postcondition
User has a clear picture of the training environment's current data state.
